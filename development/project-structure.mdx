## Overview

AlXuraan follows a feature-based folder structure organized by functionality, making it easy to locate and maintain code. The project uses React Native with Expo and follows best practices for mobile app development.

## Root Directory Structure

```
source/
├── assets/              # Static assets (fonts, images)
├── components/          # Reusable UI components
│   ├── ui/             # Base UI components (Button, Card, Theme)
│   └── *.jsx           # Feature components
├── context/            # React Context providers
├── hooks/              # Custom React hooks
├── navigation/         # Navigation configuration
├── screens/            # Screen components
├── services/           # API and external service integrations
├── store/              # Zustand state management
├── utils/              # Utility functions and helpers
├── data/               # Static data files
├── App.jsx             # Main application entry point
└── app.config.ts       # Expo configuration
```

## Entry Point

The application entry point is `App.jsx`, which sets up providers and navigation:

```jsx App.jsx
import { NavigationContainer } from '@react-navigation/native';
import { ThemeProvider } from './context/ThemeContext';
import { PlayerProvider } from './context/PlayerContext';
import AppNavigator from './navigation/AppNavigator';

export default function App() {
  return (
    <ThemeProvider>
      <PlayerProvider>
        <NavigationContainer>
          <AppNavigator />
        </NavigationContainer>
      </PlayerProvider>
    </ThemeProvider>
  );
}
```

## Best Practices

1. **Component Organization**: Keep components small and focused on a single responsibility
2. **File Naming**: Use PascalCase for component files (e.g., `PlayerMini.jsx`)
3. **Service Layer**: All API calls should go through services, never directly in components
4. **Custom Hooks**: Extract reusable logic into custom hooks in the `hooks/` directory
5. **State Management**: Use Context for UI state, Zustand for persistent data
6. **Navigation**: Keep navigation logic in navigator files, not in screens

## Adding New Features

When adding a new feature:

1. Create screen components in `screens/`
2. Add reusable components to `components/`
3. Create a navigator in `navigation/` if needed
4. Add API integration in `services/`
5. Create custom hooks in `hooks/` for data fetching
6. Update the appropriate navigator to include new screens

<Info>
  Follow the existing patterns in the codebase to maintain consistency and readability.
</Info>
